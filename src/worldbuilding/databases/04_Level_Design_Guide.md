# 레벨 디자인 활용 가이드

이 문서는 Nexus Multiverse 데이터베이스를 실제 레벨 디자인에 적용하는 방법을 안내합니다.

---

## 1. 맵 제작 시작 플로우차트

```
1. 맵 컨셉 결정
   "어떤 맵을 만들고 싶은가?"
   ↓
2. 장르 선택 (01_Genre_Database.md)
   액션? → Neon Sprawl, Void Expanse
   탐험? → Lumina Realm, Aether Empire
   생존? → Dead Zones, Ashen World
   ↓
3. 캐릭터 선택 (02_Characters_Database.md)
   플레이 스타일에 맞는 주인공 3C 확인
   ↓
4. 시나리오 선택 (03_Scenarios_Database.md)
   캐릭터의 시나리오 2개 중 선택
   ↓
5. 레벨 디자인 시작
   ↓
6. 이 가이드의 체크리스트 활용
```

---

## 2. 레벨 디자인 프로세스 통합

Tim Does Level Design의 프로세스와 Nexus 데이터베이스를 결합합니다.

### Step 1: 버블 다이어그램 (공간 설계)

1. 시나리오의 Act 구조 확인 (Act 1, 2, 3)
2. 각 Act를 버블(공간)로 표현
3. 연결 관계 설정

**예시**: Kira Chen - Data Heist

- Bubble 1: 빌딩 외부 (입구 찾기)
- Bubble 2: 환기구 경로 (스텔스)
- Bubble 3: 서버룸 (해킹 퍼즐)
- Bubble 4: 탈출 경로 (액션)

### Step 2: 색상 및 도형 적용 (Max Pears)

1. Genre Database에서 색상 팔레트 확인
2. 도형 이론 적용
   - 안전 지역: 원형
   - 위험 지역: 삼각형
   - 이동 경로: 사각형

### Step 3: 블록아웃 (3C 고려)

1. Character Database에서 3C 특성 확인
   - Camera: 1인칭? 3인칭?
   - Character: 점프력, 이동 속도
   - Controls: 특수 능력
2. 공간 스케일 조정
   - 점프력 높음 → 수직 구조
   - 스텔스 캐릭터 → 숨을 곳 다수

### Step 4: 페이싱 설계 (Sakurai Risk & Reward)

1. High 구간: 전투, 위험
2. Low 구간: 탐험, 휴식
3. 교차 배치

## 3. 세계관별 필수 레벨 디자인 요소

### Neon Sprawl (사이버펑크)

**필수 요소**:

- [ ] 수직 구조 (빌딩 내외부)
- [ ] 해킹 가능 터미널 (최소 3개)
- [ ] 카메라/드론 (감시 시스템)
- [ ] 환기구 또는 대체 경로
- [ ] 네온 조명 (색상: 블루/핑크)

**랜드마크**: 중앙 타워 (항상 보임)
**브레드크러밍**: 케이블, 홀로그램

---

### Lumina Realm (하이 판타지)

**필수 요소**:

- [ ] 넓은 전투 공간 (다수 적)
- [ ] 마법 회복 지점 (성수, 마나 샘)
- [ ] 고대 퍼즐 (최소 1개)
- [ ] NPC 아군 또는 마을
- [ ] 금색/흰색 조명

**랜드마크**: 성 또는 탑
**브레드크러밍**: 빛나는 보석, 요정의 빛

---

### Umbra Wastes (다크 판타지)

**필수 요소**:

- [ ] 모닥불 안전 지대 (체크포인트)
- [ ] 숏컷 (보스 후 개방)
- [ ] 즉사 함정 (경고 필요)
- [ ] 환경 위험 (낭떠러지, 가시)
- [ ] 낮은 채도 색상 (회색/핏빛)

**랜드마크**: 저주받은 성
**브레드크러밍**: 핏자국, 시체

---

### Ashen World (핵전쟁 후)

**필수 요소**:

- [ ] 엄폐물 (Half/Full Cover)
- [ ] 방사능 지역 (시간 제한)
- [ ] 물자 수색 포인트
- [ ] 벙커 (안전 지대)
- [ ] 재색/갈색 색상

**랜드마크**: 폐허 빌딩
**브레드크러밍**: 발자국, 피난 표지판

---

### Dead Zones (좀비)

**필수 요소**:

- [ ] 바리케이드 재료
- [ ] 소음 함정 (좀비 유인)
- [ ] 좁은 안전 경로
- [ ] 좀비 무리 (호드)
- [ ] 녹색/갈색 색상

**랜드마크**: 라디오 타워
**브레드크러밍**: 핏자국, X 표시

---

### Aether Empire (스팀펑크)

**필수 요소**:

- [ ] 움직이는 톱니바퀴 플랫폼
- [ ] 증기 분출구 (타이밍)
- [ ] 글라이더 활공 구간
- [ ] 발명 재료 수집
- [ ] 구리/황동 색상

**랜드마크**: 시계탑
**브레드크러밍**: 파이프, 증기

---

### Void Expanse (우주)

**필수 요소**:

- [ ] 무중력 구간
- [ ] 산소 관리 (시간 제한)

- [ ] 외계 환경 (다양한 행성)
- [ ] 검정/보라 색상

**랜드마크**: 행성 또는 함선
**브레드크러밍**: 홀로 내비, 조명 라인

**필수 요소**:

## 4. 실전 체크리스트

맵 제작 시 다음을 확인하세요:

- [ ] 세계관의 색상 팔레트 적용
- [ ] 도형 이론 적용 (안전=원, 위험=삼각형)

### 게임플레이

- [ ] 시나리오 Act 구조 반영 (3막 구조)
- [ ] 리스크와 리턴 명확 (Sakurai 원칙)
- [ ] 페이싱 (High → Low 교차)

### 레벨 디자인 기법 (Tim Does)

- [ ] 글로벌 랜드마크 (멀리서 보임)
- [ ] 로컬 랜드마크 (각 구역)

- [ ] 프레이밍 (중요 장면)
- [ ] 퍼널 앤 리빌 (좁음 → 넓음)

### 환경 스토리텔링

- [ ] 명시적 스토리 (텍스트, 대화)
- [ ] 암묵적 스토리 (환경 단서)

## 5. 케이스 스터디: Kira Chen - Data Heist

### 선택된 요소

- **세계관**: Neon Sprawl (사이버펑크)
- **캐릭터**: Kira "Ghost" Chen
- **시나리오**: S01 - Data Heist
- **3C**: 3인칭, 높은 점프력, 벽타기

### 공간 설계

**Act 1: 침투 (스텔스 구간)**

- 공간: 빌딩 외부 → 환기구
- 색상: 어두운 블루, 네온 핑크 강조
- 도형: 삼각형 빌딩 실루엣
- 랜드마크: 중앙 메가코프 타워 (먼 곳)
- 브레드크러밍: 환기구 그릴 (빛남)

**레벨 디자인**:

- 높이: 50층 빌딩
- 수직 경로: 외벽 파쿠르 + 환기구
- 카메라 5대 (패턴 학습)
- 페이싱: Low (탐색, 경로 계획)

**Act 2: 해킹 (퍼즐 구간)**

- 공간: 서버룸
- 색상: 차가운 블루 조명, 깜빡이는 서버 불빛
- 도형: 사각형 서버 랙
- 해킹 퍼즐: 3단계 (난이도 증가)

**레벨 디자인**:

- 넓이: 중형 공간
- 서버 랙: 엄폐물로 활용 가능
- 경비 로봇: 2대 (순찰)
- 페이싱: Medium (긴장 + 퍼즐)

**Act 3: 탈출 (액션 구간)**

- 공간: 건물 전체 (역방향)
- 색상: 경보등 빨강
- 적: 경비 8명, 드론 3대
- 탈출: 옥상 → 글라이더

**레벨 디자인**:

- 타임 어택: 3분
- 다중 경로: 정면 vs 환기구
- 최종 점프: 건물 간 대시
- 페이싱: High (액션, 추격)

### 결과물 특징

✅ 3C 활용 (벽타기, 높은 점프)
✅ 색상 팔레트 (블루/핑크)
✅ 페이싱 (Low → Medium → High)
✅ 환경 스토리텔링 (메가코프 표어, 사무실 상태)
✅ 리스크/리턴 (감시 vs 고액 보수)
